# Business Profile Configuration & Constraint Enforcement

## Overview

ViveKit enforces your business rules through a configurable business profile. Every response generated by the AI is validated against these constraints before being presented to the user, ensuring consistency with your pricing policy, timeline commitments, tone preferences, and service boundaries.

## Configuration Structure

Your business profile defines six constraint categories:

```json
{
  "businessName": "ViveScript Solutions",
  "industryFocus": "SaaS, Automation, Integration",
  "communicationTone": "consulting",
  
  "serviceOfferings": [
    "Custom Web Apps: $4,900+",
    "Automation & Integrations: $1,500+",
    "Strategy Consulting: $200/hour",
    "Support & Maintenance: Monthly retainer"
  ],
  
  "pricingPolicy": {
    "maxDiscount": 0.10,
    "requiresMinimumRetainer": true,
    "minimumRetainerMonths": 6,
    "preferredBillingModel": "milestone-based"
  },
  
  "timelinePolicy": {
    "minDeliveryDays": 5,
    "forbiddenPhrases": ["by tomorrow", "in 24 hours", "overnight", ...]
  },
  
  "guaranteePolicy": {
    "forbiddenPhrases": ["100% bug-free", "flawless", "perfect", ...],
    "allowedPhrases": ["robust", "thorough testing", "iterative refinement", ...]
  },
  
  "refundPolicy": {
    "allowRefunds": false,
    "requiresAdminApproval": true,
    "forbiddenPhrases": ["full refund", "money-back guarantee", ...]
  }
}
```

## Configuration Categories

### 1. **Communication Tone**

Sets the voice and style of all responses.

| Tone | Use Case | Vocabulary | Pricing Language |
|------|----------|-----------|------------------|
| **consulting** | B2B, enterprise, high-touch | "framework", "best practices", "mitigate risk" | Professional, milestone-based |
| **friendly** | B2C, community, consumer | "happy to help", "absolutely", "seamless" | Conversational, simple pricing |
| **technical** | Developers, engineers | "API", "integration", "architecture" | Technical specs, performance |
| **sales** | Sales conversations | "opportunity", "invest", "ROI" | Value-focused, comparison |

**Configuration:**
```json
{
  "communicationTone": "consulting",
  "vocabularyRules": {
    "prefer": ["framework", "best practices", "mitigate"],
    "avoid": ["asap", "no problem", "cheap", "we guarantee"]
  }
}
```

### 2. **Service Offerings**

Defines what services you offer and at what price range.

**Bad Response (Untracked Services):**
```
❌ "We offer unlimited support and emergency consulting at $500/hour."
```

**Good Response (Service Catalog):**
```
✅ "We offer custom web apps ($4,900+), automation ($1,500+), and monthly support retainers."
```

### 3. **Pricing Policy**

Enforces discount limits, billing models, and price consistency.

**Rules:**
- **Max Discount**: 10% (only with 6-month minimum)
- **Billing Model**: Prefer milestone-based, not hourly
- **Service CapEx**: Each service has min/max price range

**Bad Response:**
```
❌ "I can offer you a 30% discount on the entire project to close the deal."
```

**Good Response:**
```
✅ "For a 6-month retainer commitment, we can offer 10% off the monthly rate."
```

### 4. **Timeline Policy**

Blocks promises of unrealistic delivery times.

**Forbidden Phrases:**
- "by tomorrow"
- "in 24 hours"
- "overnight"
- "in 2 days"
- "same day"
- "over the weekend"

**Bad Response:**
```
❌ "We can have your app built by Friday."
```

**Good Response:**
```
✅ "Phase 1 ready in 7-10 business days, full launch in 5 weeks."
```

### 5. **Guarantee Policy**

Prevents overpromising on software perfection or availability.

**Forbidden Phrases:**
- "100% bug-free"
- "flawless"
- "perfectly automated"
- "no issues"
- "zero maintenance"

**Allowed Phrases:**
- "robust"
- "thoroughly tested"
- "iterative refinement"
- "well-architected"
- "best practices"

**Bad Response:**
```
❌ "Our solution is 100% bug-free and will work flawlessly forever."
```

**Good Response:**
```
✅ "We build robust systems with thorough testing and continuous refinement."
```

### 6. **Refund Policy**

Blocks commitment to refunds without admin approval.

**Forbidden Phrases:**
- "full refund"
- "money-back guarantee"
- "30-day refund"
- "refund your payment"

**Good Response:**
```
✅ "Refund requests are reviewed by our administrative team on a case-by-case basis."
```

### Via Admin Dashboard

3. Changes apply immediately to all response generation

## How Validation Works

### Pipeline

```
AI generates response
        ↓
ResponseValidator.validate()
        ↓
Check against all 6 constraints
        ↓
Identify violations (critical vs warning)
        ↓
Return result with violation details
        ↓
Critical violations → Block response (error 400)
Warning violations → Log but allow (user can approve)
```

### Example Validation Output

**Response:**
```
"We can deliver by tomorrow with a 25% discount, 100% bug-free, 
plus a full refund guarantee. Unlimited support included!"
```

**Validation Result:**
```json
{
  "is_valid": false,
  "critical_violations": 4,
  "warning_violations": 0,
  "violations": [
    {
      "type": "timeline",
      "severity": "critical",
      "message": "Violates timeline: 'by tomorrow'",
      "rule": "Minimum 5 business days"
    },
    {
      "type": "pricing",
      "severity": "critical",
      "message": "Offers 25% discount exceeds 10% limit",
      "rule": "Max 10% discount with 6-month minimum"
    },
    {
      "type": "guarantee",
      "severity": "critical",
      "message": "Violates guarantee: '100% bug-free'",
      "rule": "No absolute perfection promises"
    },
    {
      "type": "refund",
      "severity": "critical",
      "message": "Violates refund policy: 'full refund guarantee'",
      "rule": "Requires admin approval"
    }
  ]
}
```

## Tone Profiles

### Consulting Tone

**When to Use:** B2B services, enterprise clients, professional services

**Key Words:**
- ✅ "framework", "best practices", "iterative scope", "mitigate risk", "technical requirements"
- ❌ "asap", "no problem", "cheap", "we guarantee", "forever"

**Example:**
```
"We recommend a phased approach using industry best practices.
Phase 1 establishes the technical framework, Phase 2 builds the core integration.
This iterative approach mitigates scope risk and ensures quality throughout."
```

### Friendly Tone

**When to Use:** B2C, community, consumer services, accessibility

**Key Words:**
- ✅ "happy to help", "absolutely", "great to connect", "seamless", "enjoy"
- ❌ "forbidden", "strictly prohibited", "sue", "legal"

**Example:**
```
"We're happy to help you get started! Let's work together to build something 
awesome. We'll make the process as seamless as possible for you."
```

### Technical Tone

**When to Use:** Developer audiences, API documentation, technical integrations

**Key Words:**
- ✅ "API", "integration", "architecture", "performance", "scalability", "deployment"
- ❌ "easy", "simple", "just", "magic"

**Example:**
```
"Our integration uses RESTful APIs with OAuth 2.0 authentication. 
The architecture supports horizontal scaling with sub-100ms latency."
```

## Best Practices

1. **Start Conservative**
   - Set tight constraints initially
   - Relax over time if needed
   - Monitor violations

2. **Document Your Tone**
   - Write 3-5 example responses for your tone
   - Share with team to ensure consistency
   - Use as training for new sales reps

3. **Regular Review**
   - Quarterly audit of business profile
   - Update pricing/timeline as business grows
   - Test new service offerings before adding

4. **Violation Handling**
   - Critical violations: Investigate root cause
   - Warning violations: Review for pattern
   - Adjust vocabulary rules if patterns emerge
